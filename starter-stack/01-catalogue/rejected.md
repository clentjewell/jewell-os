## Rejection criteria

A tool is rejected (not merely parked) if it meets any of:

- **Archived** — the upstream repo is archived or unmaintained.
- **Licence-incompatible** — the licence blocks the intended use (e.g. forbids commercial use,
  requires source disclosure Jewell cannot make, or conflicts with a client contract).
- **Unsafe for shared workspace** — the tool requires broad, unscoped credentials or behaviour that
  cannot be made safe for a shared Jewell/Maxxim environment.
- **Duplicative of a better official tool** — an equivalent first-party or clearly superior
  alternative already covers the same need (e.g. a third-party MCP server duplicating an official
  vendor MCP server).
- **Too heavy for benefit** — the setup, maintenance, or review burden is disproportionate to the
  problem it solves for Jewell's current scale.
- **Raises token usage without clear benefit** — the tool inflates context or token spend with no
  corresponding gain in output quality or capability.
