# Weekly washback template

The fill-in form for the ritual described in `../operating-rhythm/weekly-washback.md`. Copy this
into the week's washback note.

## Template

```
WEEKLY WASHBACK — week of <YYYY-MM-DD>

Signals collected

| Source                         | Date       | Signal                                       |
|---------------------------------|------------|-----------------------------------------------|
|                                  |            |                                                |

Proposed changes

| Proposal | Kind (memory/SOP/strategy) | Evidence | Confidence (fact/assumption) | Tier | Recommended default |
|----------|-----------------------------|----------|-------------------------------|------|-----------------------|
|          |                             |          |                                |      |                        |

Decisions needed from Clent
-

Approved → routed where
-

Rejected → reason
-

What changed, what mattered, what can be retired (one line each)
- Changed:
- Mattered:
- Can be retired:
```

## Notes on filling it in

- **Signals collected** should cite where each signal came from (a Deepen note, a repeated
  question, a piece of friction) and when. A signal with no source is not usable evidence.
- **Proposed changes** must each carry a kind, evidence, a confidence level, and a recommended
  tier. This is the same evidence bar as `memory-promotion-rules.md`; do not bring a proposal here
  without it.
- **Decisions needed from Clent** is the list Clent actually works through. Keep it short and
  specific, one decision per line.
- **Approved → routed where** records exactly which memory layer or `jewell-os` file the approved
  item landed in. "Approved" without a destination is not finished.
- **Rejected → reason** keeps a rejected idea from being silently re-proposed. The reason is the
  point, not just the rejection.
- **The one-line summary** at the bottom is what a busy reader takes away if they read nothing
  else. Keep each line to one line.

**Next:** file the completed washback per `../operating-rhythm/weekly-washback.md`, and carry
approved items into `06-memory` and any playbook file they touch.
