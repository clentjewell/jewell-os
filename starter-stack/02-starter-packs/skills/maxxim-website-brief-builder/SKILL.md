# Maxxim website brief builder

Turn approved Design outputs into a build-ready website brief.

## When to use

Once Design outputs for a website are approved, and before a build begins, so the build team
starts with everything it needs.

## Inputs

- Approved Design outputs from `05_Approved`: positioning, messaging, offers, measurement plan.
- Any brand assets and technical constraints the engagement owner has already confirmed.

## Outputs

- A build-ready website brief: audience, messaging, sitemap, page-by-page briefs, and a
  measurement plan.

## Steps

1. Read the approved Design outputs only. Do not draft a brief from un-approved Design work.
2. Draft the audience and messaging summary from the approved material.
3. Draft the sitemap, then a page brief for each page: purpose, key messages, calls to action,
   and any content or asset needs.
4. Draft the measurement plan: what is tracked, and against which goal.
5. **Approval gate.** The engagement owner reviews the brief for accuracy against the approved
   Design outputs and approves it before it is handed to the build team.

## Guardrails

- Must only draw from approved Design outputs, never a draft.
- The brief template itself belongs to Maxxim as reusable structure. The filled brief is
  client-specific and must never enter the Maxxim template library unsanitised.
- This is a Tier 1 internal handover document. Approved by the engagement owner before it reaches
  the build team. It only becomes Tier 2 if it is shared with the client directly.

## Where the output is filed

`02_Design`, alongside the approved Design outputs it was built from.

## Example

Acme Co's approved messaging names three audience segments and a core offer. The skill drafts a
six-page sitemap (home, offer, about, case studies, contact, resources), a page brief for each,
and a measurement plan tracking enquiry form completions against the offer page. The engagement
owner reviews and approves before the build team starts.

## Success measure

The build starts with no back-and-forth: the build team has everything it needs in the brief.
