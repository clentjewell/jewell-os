# 09-playbook

The delivery playbook: the 3D Process spine and how the Ask and portal layers behave. Full detail

and [Section 9](../OPERATING-SYSTEM.md#section-9-ask-jewell-ai-and-client-portal-layer).

## The delivery spine

Every engagement runs on the 3D Process:

1. **Discover** — capture context once, via the ABC front door and guided discovery. Never ask
   what the system already knows.
2. **Design** — draft the artifact set in the background, as gated drafts.
3. **Deploy** — move approved work live and into the portal; track in Asana.
4. **Deepen** — capture signals continuously.

## The one hard rule

Day-to-day noise must not overwrite long-term strategy. Learnings wash back only through weekly,
monthly or quarterly review, with human approval.

## Ask Jewell AI and portals

Guided, not generic. Approved sources only. Cite the source. Say when unsure. Escalate. Treat
user types differently. Never cross clients. Never invent scope, pricing, proof or strategy.

The playbook prose, the stage checklists, the gate definitions, and the Ask/portal behaviour
rules by role. Client work itself lives in Drive, not here.
