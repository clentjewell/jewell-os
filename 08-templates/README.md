# 08-templates

Reusable templates: the structures the routines and skills fill in. Templates are structure, never
client-specific content. A sanitised, proven template can become Maxxim IP (Section 2).

## What belongs here

- Today door template.
- Meeting-to-actions note template.
- Client intelligence brief template.
- 3D Discover / Design / Deploy / Deepen pack templates.
- Weekly pack, finance pack, month-end pack templates.
- Case study template (sanitised, consent required before publishing).
- Website brief template.
- Proposal-to-delivery handover checklist.

## What does not belong here

Any client's actual data. Templates carry placeholders, not real names, figures or strategy.

## Rules

- One template per artifact. Do not fork the same template per client.
- Keep templates in the house voice: plain, British English, ends on "Next".
- When a template proves itself in delivery, propose it for the Maxxim IP library after
  sanitisation and review.
