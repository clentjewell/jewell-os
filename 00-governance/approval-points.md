# Approval points

Where a human must decide before anything happens. This is the gate list. If an action is on it,
the system proposes and stops until a named human approves.

## Decision tiers

- **Tier 1 — Ronnie's call.** Internal, reversible, no client or legal exposure.
- **Tier 2 — default and approve.** The system proposes a recommended default; Clent approves it
  in one word.
- **Tier 3 — Clent alone.** External sends, credentials, and anything irreversible. The detailed
  thresholds live in the private layer, not here.

## No longer gated — the standing improvement directive (Clent, 10 July 2026)

Promoting learnings into memory, SOPs, playbooks and docs is pre-approved on an ongoing basis:
anything that improves the system may be executed directly, logged in the decision log, and kept
reversible. Clent can veto or unwind any promotion, and the veto is logged. (This removed the
former "Promote anything into memory — Clent" gate.) Long-term strategy is the one exception —
washing a learning back into strategy stays Clent's call, below.

## Always needs approval

| Action | Approver |
| --- | --- |
| Wash a learning back into long-term strategy | Clent |
| Change pricing or scope | Clent (with engagement owner) |
| Make or change a client commitment | Engagement owner + Clent |
| Publish a client-facing output | Engagement owner |
| Add an item to the Maxxim IP library | Clent + review |
| Change a permission, credential, scope or authentication | Clent |
| Grant or widen access | Clent |
| Delete anything | Clent (archive first)¹ |
| Move personal or legal content | Clent only |
| Change a governance rule or the constitution | Clent |
| Change an `AGENTS.md` or portal answering rule | Clent |

¹ Clent may approve direct deletion as a recorded exception, where archiving cannot stop the harm
(for example, recurring tasks that respawn on completion).

## Never automated

- Granting access.
- Rotating a live client credential.
- Changing a client-facing permission.
- Sending anything to a client.
- Publishing anything public.

## How a proposal is presented

State the kind (recommendation), the change, the reason, the risk, and the smallest safe version.
Then stop. Do not act until a named approver signs off, and log the decision when they do.
